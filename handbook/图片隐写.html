# 1、wiki-某账号信息中存放的flag

题目地址：http://hackattack.cn:49213

​ 题目给的地址是可以看见这里是要登录后才有功能，并且没有验证码，第一思路就是弱口令，不过一顿弱口令爆破之后无法进入，那应该就是别的方法进入

既然弱口令尝试了无法进入，但是可以看到除了登录还剩一个功能点，也就是注册。盲猜可能是越权，越权就是利用一些方式绕过权限检查，访问或者操作其他用户或者更高权限。那我们就通过注册普通用户越权到管理员用户 先随便注册一下

可以看到我们刚刚新建的用户建立成功，然后就是正常的登录进去

进来后看到提示 有一个zip文件下载下来居然报毒了，里面肯定有马

果不其然是一句话木马，看到里面还有提示路劲，但是不知道路径也无法利用起来。还是先去抓包分析一下数据包看看有没有什么新发现

抓包之后发现这个网站的 Cookie 的值有些奇怪，看起来很像base64编码。解码看看

可以看到解码后的信息不就是我创建的用户？可以看到这个 Cookie 的用户名被双层加密了分别是 URL编码 > Base64编码 而用户名后面的那一串乱 码我们不知道是什么，但很大概率只要把用户名改成 admin 然后跟上后面的 = 号部分的内容拼接一下就能越权 到管理员用户

**现在先把 admin 用base64 编码一下，得到 YWRtaW4= **

然后把原本的dGVzdDEyMw =|0|SHE2ltbjREhhfODdVvqzy432CjUPraIIaWXWCAf0IO8= 这个 也就是原先的 Cookie 把 = 号前面的内容替换成 Base64 编码后的 admin

YWRtaW4=|0|SHE2ltbjREhhfODdVvqzy432CjUPraIIaWXWCAf0IO8=

到这里还没结束因为这里还有一个可疑参数 |0| 等号前面的内容代表用户名，那这个数字肯定就是代表权 限，我们把他改成 1 然后再抓包替换一下就可以看到现在我们的权限就越权变成了管理员用户

YWRtaW4=|1|SHE2ltbjREhhfODdVvqzy432CjUPraIIaWXWCAf0IO8=

那么此时我们就拿着这个 Cookie 在登录界面刷新一下，然后替换掉数据包中本身的 Cookie 即可完成越 权操作就可以进入管理员用户的账号了

随后经过一番尝试发现只有管理里面的上传点知道文件上传路径，其他地方即使传上去了也不知道传去哪里 了。

如下 http://hackattack.cn:49213/doku.php?id=welcome&do=admin&page=extension&tab=install

可以看到在这里上传的压缩包会被服务器自动解压到指定目录 /lib/plugins/packagename/，于是我就 写了一个1.php 压缩成 1.zip 之后上传 根据提示可以知道我的木马被解压到了/lib/plugins/1/1.php 拼接访问后即可getshell

http://hackattack.cn:49213/lib/plugins/e/e.php

flag在网站的目录里翻翻就能看见了

flag{hdb2021forphpwiki}

# 立春橘-貌似某用户的用户名是flag

题目地址：http://hackattack.cn:49215

来到主页是这样的，是一个个人的博客站点

右上角有一个显眼的登录按钮，老样子试一下常见的弱口令

admin admin123

多试试了几个就进来了

题目提示的某用户 ，肯定有东西 既然是用户肯定直接看用户管理，果然就看到了flag

flag{zhss-caos-u98c-d6cz}

python dirsearch.py -u "http://hackattack.cn:49215" -e*是扫全部的意思

python dirsearch.py -u "http://hackattack.cn:49215" -e proxy "http://127.0.0.1"

扫描完之后发现了一个神奇的东西，疑似网站源码泄露

可以发现这就是源码泄露，这里还有一个数据库文件，可惜里面没有有价值的信息。 不过在这里可以看见这些代码的意思应该是生成一个验证码 code 一开始没想到可以通过这个方法注册低 权限用户，我直接请求郑总出手。随后他根据这个源码泄露的路径在网站上拼接访问如下

http://hackattack.cn:49215/zc/phone.php

这里返回的数据是 json 数据我们只需要看 code 是多少就行了，这个就是验证码

然后来到注册界面注册一下账号

这里的验证码一定要填写刚刚那个地方的验证码才能注册成功

**然后返回登录看看有这个账号有什么新东西 里面一样可以上传文件，但毫无意外的是都被拦了 **

不过却很意外的发现这里多了一个管理员没有的功能

一般看见这种命令执行的地方就很有可能是存在RCE漏洞（远程代码执行漏洞）

开启抓包然后点一下查看公共文件列表尝试一下 抓到包之后丢到 重放 模块进行测试 这里点一下页面渲染，可能看结果会更清晰

可以看到传递的是path参数，我们把他闭合然后输入其他的Linux系统命令试试

可以看见页面出现whomai的执行结果，此时可以证明这个地方存在RCE漏洞，那我们就可以通过命令执行的 方法写一个shell进入然后连接

相关命令

echo '< = eval($_REQUEST[8]); ?>' > xyc42b.php

写完之后点击发送，然后再执行一下 ls 命令查看是否写入 可以看到成功写入了

接下来再执行 pwd 命令查看当前路径，然后拼接域名之后我们就可以使用蚁剑连接了

根据pwd的执行结果可以得知我们的shell写在了服务器的这个位置，随后在浏览器访问一下看看服务器的根 目录是哪个，经过一番探测后得到下面这个路径

/getfilelist/xyc42b.php

现在可以使用蚁剑开始连接这个路径了
